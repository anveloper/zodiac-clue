0. 배경 / 방향 전환

  • 게임 주제는 확정이 아니다. "클루"는 현재 프로토타입일 뿐, 바뀔 수 있다.
  • 전략: 재사용 가능한 "게임 엔진"과 교체 가능한 "게임 콘텐츠(주제)"를 분리한다. 주제가 바뀌어도 엔진(멀티/방/NPC/딜레이/문서체계)은 그대로 재사용.

1. 게임 플레이 엔진 (확정 룰 — 주제 불문 유지)

1.1 1인도 가능한 멀티 환경 구현됨

방장 혼자여도 시작 가능, 부족 인원은 NPC 충원. Colyseus 권위 서버 + 상태동기화.

1.2 방 만들기 + 캐릭터 선택 구현됨

방 생성 / 초대 링크(/room/<코드>) / 대기실 / 호스트 시작. 캐릭터 선택은 선택 사항, 중복 검증은 방 안(대기실)에서 실시간.

1.3 룰 기반 NPC + LLM 표현 레이어 구현됨 (갱신 2026-07-21: narrate+fallback+캐시·EMA 절반딜레이·speak/helperWhisper)

  • 결정(무엇을)은 결정론적 규칙 엔진이 계산 — 이동·제안·반증·추리·판정. 환각 차단 · 무료티어 안전.
  • 대사·연출(어떻게 보이나)은 LLM 경유. 규칙 엔진이 확정한 "결정된 정보"만 입력 → 자연어 대사/제스처 생성. LLM은 진실값을 만들지 않는다(표현 전용).
  • 딜레이 = 사용자 평균 플레이 시간의 절반. 사용자 턴 소요시간을 이동평균으로 측정 → NPC 지연을 그 1/2로. (사람다운 템포 + 호출 분산 + 무료티어 절약)
  • 폴백: LLM 실패/한도(429) 시 규칙 기반 정적 대사로 대체 → 게임 안 끊김.

1.4 엔진 추상화 (게임 교체 대응) 예정

  • 게임별 요소를 GameDefinition 인터페이스로 분리: 구성물(카드/역할/맵), 승리조건, NPC 결정 정책(decideTurn), 초기 배치, 표현 라벨/이모지.
  • 엔진 코어(방/턴/동기화/NPC 스케줄/LLM 표현/딜레이/재접속)는 게임 독립.
  • 현재 "클루"는 첫 GameDefinition 구현으로 이관. 새 주제 = 새 GameDefinition만 작성.

2. AI 개발 협업 규칙 (문서 체계)

2.1 plans 워크플로우 (active → done)

  • 작업 단위 = docs/plans/active/<task>.md (압축 체크리스트).
  • 완료 시 docs/plans/done/으로 이동(git mv).
  • CLAUDE / CODEX 등 모든 AI 도구가 동일 문서 기반으로 작업 → CLAUDE.md와 AGENTS.md가 같은 규칙 참조.

2.2 문서 이원화 (.md = AI / .html = 사람)

  • .md (AI 전용): 읽기 토큰 절약이 목적 — 다른 에이전트 참조 시 낭비 없도록 핵심만 밀도 높게.
  • .html (사람용): Claude가 매번 별도로 "한눈에 보기 좋게" 디자인해 작성. md→html 자동 변환/빌드는 쓰지 않는다 (기계 변환은 가독성·정보 위계 미달).
  • docs/plans/*는 .md만(AI 전용). 그 외 문서는 .md(압축) + .html(디자인) 2중.

2.3 문서 공개 (추후)

도메인 구매 후 도메인/docs에 .html 개발 문서 공개(작업자용). .html은 자체 완결(inline CSS)로 정적 서빙만으로 열람 가능.
