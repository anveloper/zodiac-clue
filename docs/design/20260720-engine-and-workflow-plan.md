## 0. 배경 / 방향 전환

- **게임 주제는 확정이 아니다.** "클루(Cluedo)"는 현재의 프로토타입 구현일 뿐, 주제는 바뀔 수 있다.
- 전략: **재사용 가능한 "게임 엔진"과 교체 가능한 "게임 콘텐츠(주제)"를 분리**한다.
  주제가 바뀌어도 엔진(멀티/방/NPC/딜레이/문서체계)은 그대로 재사용한다.

## 1. 게임 플레이 엔진 (확정 룰 — 주제 불문 유지)

### 1.1 1인도 가능한 멀티 환경  — [상태: 구현됨]
- 방장 혼자여도 시작 가능. 부족 인원은 NPC로 충원. Colyseus 권위 서버 + 클라 상태동기화.

### 1.2 방 만들기 + 캐릭터 선택  — [상태: 구현됨]
- 방 생성 / 초대 링크(`/room/<코드>`) / 대기실 / 호스트 시작.
- 캐릭터 선택(선택 사항), **중복 검증은 방 안(대기실)에서** 실시간.

### 1.3 룰 기반 NPC + LLM 표현 레이어  — [상태: 일부 구현 / 확장 필요]
- **결정(무엇을)은 결정론적 규칙 엔진**이 계산한다: 이동·제안·반증·추리·판정.
  → 환각 차단, 무료티어 안전(진실값을 LLM이 만들지 않음).
- **대사·행동 연출(어떻게 보이나)은 LLM 경유**: 규칙 엔진이 확정한 **"결정된 정보"**만 입력으로,
  LLM이 **자연어 대사/제스처**를 생성한다. LLM은 게임의 진실값을 생성/변경하지 않는다(표현 전용).
- **딜레이 = 사용자 평균 플레이 시간의 절반**: 사용자 턴 소요시간을 이동평균으로 측정 →
  NPC 행동 지연을 그 **1/2**로 둔다. (사람다운 템포 + 호출 분산 + 무료티어 절약)
- **폴백**: LLM 실패/한도(429) 시 **규칙 기반 정적 대사**로 대체 → 게임이 끊기지 않는다.

### 1.4 엔진 추상화 (게임 교체 대응)
- 게임별 요소를 `GameDefinition` 인터페이스로 분리한다:
  - 구성물(카드/역할/맵), 승리조건, **NPC 결정 정책**(decideTurn), 초기 배치, 표현용 라벨/이모지.
- **엔진 코어**(방/턴/동기화/NPC 스케줄/LLM 표현/딜레이/재접속)는 **게임 독립**.
- 현재 "클루"는 **첫 GameDefinition 구현**으로 이관한다. 새 주제 = 새 GameDefinition만 작성.

## 2. 게임 외적 세팅 — AI 개발 협업 규칙 (문서 체계)

### 2.1 plans 워크플로우 (active → done)
- 작업 단위 = `docs/plans/active/<task>.md` (압축 체크리스트).
- 완료 시 `docs/plans/done/`으로 이동(`git mv`).
- **CLAUDE / CODEX 등 모든 AI 도구가 동일 문서를 기반으로 작업**한다 →
  `CLAUDE.md`와 `AGENTS.md`가 **같은 규칙(this workflow)**을 참조(single-source).

### 2.2 문서 이원화 (.md = AI / .html = 사람)
- `docs/plans/*` = **AI 전용, 압축된 `.md`**.
- 그 외 문서(설계/기능/가이드) = **`.md` + `.html` 2중 작성**.
  - AI는 `.md`(압축)를 읽고, 작업자(사람)는 `.html`로 기능을 확인한다.

### 2.3 문서 공개 (추후)
- 도메인 구매 후 **`도메인/docs`에 `.html` 개발 문서 공개**(작업자용).
- `.html`은 자체 완결(inline CSS)로 작성해 정적 서빙만으로 열람 가능하게 한다.
