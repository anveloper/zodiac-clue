VM 레시피 (Oracle / EC2 공통)

  1. Node 20 설치(nvm 또는 nodesource), corepack enable로 pnpm 활성.
  2. 레포 클론 → pnpm i → 서버 실행(pnpm --filter @zodiac-clue/server dev 또는 build 후 node/tsx).
  3. apps/server/.env에 GEMINI_API_KEY·GEMINI_MODEL·PORT=2567 — 커밋 금지 유지.
  4. Caddy 리버스 프록시(자동 Let's Encrypt) — :443에서 wss 종단:
game.example.com {
    reverse_proxy localhost:2567
}
  1. pm2 또는 systemd로 Node 상시 유지 + 재부팅 자동 기동.
  2. 방화벽/보안그룹: 443, 22(SSH)만 개방. 2567은 외부 비공개(프록시 경유).

클라 연결 설정

  • 서버 주소를 env로 분리: 개발 ws://localhost:2567, 운영 wss://game.example.com.
  • 클라를 https로 배포하면 반드시 wss — 브라우저가 ws:// 혼합콘텐츠 차단.
  • 서버 CORS/allowed origins에 클라 도메인 등록.

체크리스트

  • 서버 호스트 선택(Oracle / EC2 / Fly)
  • 도메인·서브도메인 + DNS A레코드
  • Caddy 자동 TLS 동작(wss 접속 확인)
  • pm2 / systemd 상시 기동 + 재부팅 복원
  • .env 시크릿 주입(키 커밋 금지)
  • 클라 운영 빌드에 wss 주소 주입 + 배포
  • 방화벽 443 / 22만 개방

주의

  • 단일 인스턴스 = 재시작 시 진행 중 방 소실(MVP 허용). 영속 필요 시 이후 Redis/드라이버.
  • 키·시크릿은 호스트 env/secret로만.

OCI 특유 함정(겪은 것)

  • 홈 리전 영구 고정: 무료 계정엔 한국 리전이 선택지에 없을 수 있음 → Osaka(ARM 확보에도 유리).
  • 네트워킹은 VCN 마법사로: 인스턴스 마법사의 인라인 서브넷은 공인 IP 토글이 잠기는 버그. Networking → VCN Wizard → Create VCN with Internet Connectivity로 먼저 만들고 인스턴스에서 기존 VCN 선택.
  • 방화벽 2겹: OCI Security List(Ingress 80·443) + OS iptables(80·443 + 영구저장) 둘 다 열어야 함.
  • ARM 용량: A1.Flex 수시로 out-of-capacity → 재시도 또는 E2.1.Micro 우회.

재배포/업데이트

  • 서버: VM에서 git pull && pnpm i && sudo systemctl restart zodiac-server.
  • 클라: 루트에서 vercel --prod(또는 GitHub 연동 push).
