핵심 원칙 — load-bearing AI는 "표현 전용".
무엇을 하는가(결정·진실값)는 결정론적 규칙엔진이, 어떻게 보이는가(대사·연출)는 LLM이 담당한다. LLM은 정답이나 남의 패를 만들지 않는다 → 환각 차단 · 무료티어 안전 · 재현성.

2계층 구조

① 결정 계층 규칙엔진

서버 clue-room.ts. 정답 봉투·카드 분배·이동(입구/주사위/통로)·제안·반증·고발 판정이 전부 결정론.

  • NPC 추리: 후보집합에서 자기 손패·자기 제안 반증·공유 공개카드를 소거 → 남은 후보로 제안.
  • 반증 안 됨 + 자기 미보유 → 그 3장이 봉투 → 정확한 고발(봇은 오답 고발 안 함).
  • LLM 장애와 무관하게 진행·판정 항상 정상.

② 표현 계층 LLM(Gemini)

서버 ai/narrator.ts. 규칙엔진이 확정한 결정된 정보만 입력 → 사극 대사 한 문장.

  • systemInstruction으로 형식·길이·진실값 생성 금지 계약.
  • 성격(PERSONA)·말투(VOICE) 주입 → 같은 결정도 캐릭터별 말투 상이.
  • 폴백: 키없음/타임아웃/429/오류 시 규칙기반 대사(intro/outro로 캐릭터색 유지).

Gemini 운용 (무료티어 안전)

  • 모델 gemini-flash-lite-latest(~1s). flash 계열은 thinking 모델 → thinkingConfig.thinkingBudget:0으로 꺼야 빈 응답 방지.
  • maxOutputTokens:64 · temperature:0.95 · AbortController 4초 타임아웃.
  • 호출 최소화: 이벤트당 1콜 + 동일상황 캐시(LRU 200) 재사용, 프레임 루프 호출 0.
  • 키는 서버 .env(gitignore)에만 — 클라·로그·커밋 노출 없음.

사람다운 템포

NPC 행동 딜레이 = 사용자 평균 턴시간의 절반(EMA, 800~7000ms 클램프). 턴 내 2박자(방 이동 → 잠깐 쉬고 → 제안)로 흐름 인지.

검증 — 결정 vs 표현 분리

결정(규칙엔진, 동일)표현(LLM, 페르소나별)
제안 = 뱀 무녀 / 오랏줄 / 후원 (진실값 동일) 잔나비 광대(익살): "낄낄, 후원서 오랏줄이라니 아주 볼만하구먼!"
닭 훈장(훈계조): "쯧쯧, 오랏줄로 엄히 다스림이 마땅하렷다."

봇 수렴: 공유 공개카드 도입 후, 사람 전멸 뒤 봇만 남아도 유한 시간에 종결(측정 ~40s·제안 13회).

재현성 · 안전 요약

  • LLM 실패해도 게임 로직·판정 불변(폴백).
  • LLM은 정답/손패를 모름 → 치트·환각 불가.
  • 무료티어: thinking off + 캐시 + 타임아웃 + 이벤트당 1콜.
