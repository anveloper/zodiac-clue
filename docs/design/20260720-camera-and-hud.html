게임형 카메라 & 플로팅 HUD

문제

지금은 왼쪽 보드 + 오른쪽 사이드패널(단서·로그) 레이아웃이라 게임보다 대시보드처럼 보인다. 보드 전체가 뷰포트에 들어와 몰입감·긴장감이 없다.

● 파란 말 = 내 캐릭터(항상 화면 중앙 근처) ▨ = 개인 추리 메모(로컬)

요구사항

  1. 보드가 화면보다 크고 내 캐릭터 기준 탑뷰로 확대되어 보인다.
  2. 이동 시 카메라가 내 캐릭터를 부드럽게 따라온다.
  3. 휠 스크롤로 줌 인/아웃.
  4. 특수키를 누른 채 이동하면 자유시점으로 다른 영역 확인, 놓으면 내 캐릭터로 복귀.
  5. 단서 패·증거 체크·로그는 화면 모서리 플로팅 UI로만 (사이드패널 제거).

구현 접근 예정

1. 풀스크린 캔버스

Scale.FIT → Scale.RESIZE, #game을 100vw×100vh로. 사이드패널 DOM은 제거하고 플로팅 오버레이로 대체.

2. 추적 카메라 Phaser Camera

cam.setBounds(0,0,BOARD_W,BOARD_H) + cam.startFollow(myDisc, true, 0.12, 0.12). 내 토큰은 room.sessionId로 식별(현재 렌더는 전체 id 순회만 하므로 내 id 저장 추가 필요).

3. 줌 (휠)

input.on('wheel', …) → cam.setZoom(clamp(zoom×배율, MIN, MAX)). 기본 1.6~2.0배, 축소 MIN≈0.6(보드 전체 조망), 확대 MAX≈3.

4. 자유시점 (특수키 hold)

Space(또는 Alt) keydown → cam.stopFollow(), 드래그 팬 + 방향키 팬. keyup → startFollow로 부드럽게 복귀. 자유시점 중에는 캐릭터 이동(move 전송) 잠금(키 충돌 방지).

5. 플로팅 HUD (DOM 오버레이)

  • 좌하 — 내 단서 패: 받은 카드 이모지+라벨.
  • 우측 — 증거 체크리스트: 용의자/흉기/장소 전 항목 표, 클릭 토글(있음/의심/모름). 개인 추리 노트라 서버 전송 X(로컬스토리지 캐시).
  • 우하 — 기록/알림: 제안·반증·NPC 대사. 접이식.

6. 조작 안내 + 반응형

하단 얇은 바로 WASD 이동 · 휠 줌 · Space 자유시점 · 제안/고발 안내. 모바일에서 HUD 접힘/토글, 리사이즈 시 카메라 bounds·HUD 위치 재계산.

주의

  • 줌·팬은 클라 로컬 상태 — 서버/네트워크 무관.
  • 증거 체크는 개인 노트 → 서버 스키마에 넣지 않음.
  • 자유시점과 이동키 충돌 → 자유시점 활성 시 move 억제.

롤아웃 순서

풀스크린 캔버스 → 추적 카메라 → 줌 → 자유시점 → HUD(단서/증거/로그) → 조작안내 → 반응형
