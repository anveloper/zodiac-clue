## goal
플레이 기본 루프(이동·턴·NPC 배치)의 체감 버그 3종을 바로잡는다.

## 이슈 & 원인(코드 확인 완료)

### 1. 이동이 WASD로 안 됨 → 안내는 화살표로 (조치됨)
- 원인: `apps/client/src/scenes/game-scene.ts` keydown이 `e.key`("w/a/s/d") 매칭. **한글 IME에서 WASD가 자모(ㅈㅁㄴㅇ)로 들어와** 매칭 실패. 화살표는 IME 영향 없어 정상.
- 조치: **안내 문구를 화살표(↑↓←→)로 갱신** (index.html, 완료).
- 근본책(선택): 물리키 `e.code`(KeyW/KeyA/KeyS/KeyD)로 바꾸면 IME 무관하게 WASD도 동작. → 후속 여부 결정.

### 2. NPC 초기 위치가 방이 아님
- 원인: `clue-room.ts` `spawnPoint(index)`가 그리드 **모서리/가장자리**((0,0),(23,0)…)만 반환 → 전부 복도. `roomAt()=""`. `addBot()`도 이 스폰을 써서 봇이 방 밖에서 시작.
- 영향: NPC가 방에서 시작하지 않아 어색하고, 첫 턴 제안 흐름도 방 진입부터 해야 함.
- 고칠 방향: **봇은 `ROOM_REGIONS` 중 서로 다른 방 중심 좌표로 스폰**(중복 방지). 사람 스폰은 현행 유지 or 문간 배치 검토.

### 3. 턴이 무시됨 — 이동이 턴과 무관
- 원인: `handleMove()`에 **`currentTurn` 검사 없음** → 누구든(자기 턴 아니어도) 언제나 이동 가능. 제안/고발은 턴 검사가 있으나 이동은 없음.
- 부수: 사람 턴은 제안 후 자동 진행 없이 **수동 '턴 종료'** 의존 → 턴 경계가 흐릿.
- 고칠 방향(결정 필요):
  - (a) **이동을 현재 턴 플레이어로 제한** + 턴당 이동 규칙(예: N칸/이동 후 제안·종료).
  - (b) 또는 자유 이동 유지하되 **제안/고발/종료로만 턴 소비** 명확화(이동은 자유, 턴 액션만 게이팅).
  - 클라도 자기 턴 아닐 때 이동 입력 억제 + 현재 턴 UI 강조 필요.
