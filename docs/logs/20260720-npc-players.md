# 20260720 — NPC(봇) 플레이어 + 초대 참가 UX

> 사람이 6명이 안 차도 **NPC로 자동 충원**하고, NPC도 카드패를 갖고 방을 옮겨 다니며
> 제안·반증·추리를 하는 참가자로 동작하게 함. (규칙기반 봇)

---

## feat — NPC(봇) 플레이어 (`apps/server`)

- **자동 충원**: 방장이 시작하면 빈 자리를 **최대 6인까지 NPC로 채움**(`addBot`). 사람 1명 + NPC 5명도 가능.
- **카드패 보유**: NPC도 사람과 동일하게 카드를 분배받음(서버 보관). 사람의 제안을 **NPC가 반증**하고, NPC의 제안도 사람/다른 NPC가 반증.
- **턴 행동(`runBotTurn`)**: 자기 턴에 ① 임의의 방으로 이동 → ② 남은 후보로 제안 → ③ 반증받은 카드는 추리 노트에서 제거 → ④ 확신 시 고발.
- **규칙기반 추리**: `BotKnowledge`(용의자/~~흉기~~장물/장소별 남은 후보 Set). 자기 손패 제거로 초기화, 반증 카드 제거로 좁힘. "아무도 반증 못했고 내가 3장 다 안 가짐 → 그 셋이 정답" 규칙 + 후보 1개로 좁혀지면 고발. (오답 고발은 안 하도록 확신할 때만)
- **턴 스케줄**: `this.clock.setTimeout`으로 봇 턴을 ~1.6초 간격 진행(사람 턴은 입력 대기). 제안/반증/고발 로직을 사람·봇 공용(`doSuggestion`/`doAccusation`)으로 리팩터.
- 스키마 `Player.isBot` 추가.

## feat — 초대 참가 시 캐릭터 선택 선택사항화 (client)

- 랜딩에서 **캐릭터 미선택도 방 만들기/참가 가능**. 서버가 빈 자리 자동 배정 → **대기실에서 확정/변경**(중복 검증은 방 안에서).
- 초대 링크로 들어오면 "[참가] 후 대기실에서 캐릭터를 고르세요" 안내.
- 대기실: 방장은 **혼자여도 시작 가능**(빈 자리 NPC 충원). 게임 화면 토큰/라벨에 NPC는 🤖 표기.

## 검증 (agent-browser 실측)

- ✅ 캐릭터 미선택으로 방 생성 → 서버가 생쥐 서생 자동 배정
- ✅ 단독 시작 → "NPC 5명 합류" → 6인. 봇들이 방 이동·제안, 사람이 반증하는 로그 연속 확인
- ✅ 보드에 🤖 NPC 5명이 각 방에 분산 렌더, 사람 손패 표시
- ✅ 3패키지 타입체크·클라 빌드 통과

## feat — 보드 비주얼 (보드게임 스타일)

- 회색 그리드 → **클루 보드게임 룩**으로 렌더 개선(`game-scene`, Phaser 벡터, 외부 에셋 0 → 라이선스 안전).
  - 방 9칸: 한지(한옥) 바닥 + 둥근 테두리 + 다크 **명패**(방 이름).
  - 중앙 **🎁 잔치상**(클루 중앙 자리) + 금색 테두리.
  - 나무결 복도 + 옅은 그리드.
  - 말: 컬러 **원형 말 + 이모지 얼굴 + 이름표**(🤖는 NPC), **현재 턴은 금색 링**으로 강조. 같은 칸 겹침 방지용 id 기반 소량 오프셋.
