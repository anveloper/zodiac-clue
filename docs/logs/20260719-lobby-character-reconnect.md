# 20260719 — 메인 로비 흐름 (캐릭터 선택·초대 링크·대기실·세션 유지)

> 그동안 이름을 `prompt()`로 받고 곧장 공용 방에 자동 입장하던 걸,
> **"캐릭터 선택 → 방 만들기 → 초대 링크 공유 → 링크로 참가 → 대기실 → 방장 시작"**
> 하는 실제 멀티플레이 진입 플로우로 교체.

---

## feat — 십이지신 캐릭터 선택 (`packages/shared`, client)

- 이름 입력 제거. 랜딩에서 **십이지신 12캐릭터를 이모지+호칭으로 선택**.
- **호랑이는 잔치 주최자라 잠금**(후속 AI 증인/반전용) → 플레이 가능한 용의자 = 나머지 **11지신 손님**.
- 선택한 캐릭터가 곧 정체성(이름=호칭, 토큰=이모지). 로비·게임 토큰에 이모지 표기.
- shared에 `ZODIAC`(12) · `SUSPECTS`(11) · `HOST` · `EMOJI` 추가.

## feat — 방 만들기 / 초대 링크 참가 (server, client)

- `client.create("clue")` → 방 생성(방장), `client.joinById(code)` → 코드로 참가.
- **초대 링크** `?room=<roomId>` 발급 + 복사 버튼. 링크로 들어오면 코드 자동 입력.
- **대기실(로비)**: 참가자 목록(이모지·방장 표시) + 인원수. 서버에 `host` 개념 추가 — **방장만 시작** 가능, 나머지는 대기 안내.
- 시작 시 `room.lock()`으로 난입 방지. `phase === "playing"` 되면 게임 화면으로 전환.
- 제안/고발 입력은 드롭다운 모달 유지.

## feat — 새로고침 세션 유지 (탭 기준) (server, client)

- 서버 `onLeave`: 비자발적 이탈이면 `allowReconnection(60s)`으로 좌석 유지 → 재접속 시 복구.
- 클라: 접속 시 `reconnectionToken`을 **sessionStorage**(탭 단위)에 저장, 로드 시 있으면 `client.reconnect()`로 자동 복귀. 실패 시 토큰 폐기 후 랜딩.

## fix — 접속 실패 메시지

- WebSocket 연결 실패(`ProgressEvent`)를 `[object ProgressEvent]` 대신 읽을 수 있는 안내로 표시.
